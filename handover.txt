■ セクション6：スコアリングモデル（確定版）
================================================================

モデル名：アルティメットコアスキャン v4.2（一時確定）
総合スコア = Real ROIC(40%) x トレンド(35%) x 価格(25%)

変数1：Real ROIC（40%）
= ROEスコア x 60% + FCF変換率スコア x 40%
ROE基準：25%以上→100pt / 20%→85pt / 12%→58pt / 5%以下→20pt
FCR基準：80%以上→78pt以上 / 60%→62pt / 20%以下→26pt

変数2：トレンド（35%）
= ROEの3-4年傾き x 60% + FCR傾き x 40%
傾き基準：+4%/年以上→100pt / +2%→82pt / 0%→64pt / -2%→28pt

変数3：価格（25%）
= PEGスコア x 50% + FCF利回りスコア x 50%
PEG基準：0.5以下→100pt / 1.0→72pt / 1.5→42pt / 2.0超→26pt
FCF利回り：8%以上→100pt / 4%→70pt / 2%→38pt / 0%以下→8pt

ランク：S>=80 / A>=65 / B>=50 / C<50 / D=低スコア

バックテスト結果（H001-C採択）：
3年保有：超過リターン+4.46%/年（目標+3.9%達成）
上位10銘柄3年：+122.2%（日経比+85%）
5年保有：勝率100%・超過+287%

■ セクション16：重要な教訓・失敗パターン
================================================================

1. 実データを確認する前にソリューションを設計しない
2. 存在しないシート名を自信を持って述べない
3. HTMLコードを書く前に必ずgrep -nでベースHTMLを確認する
4. コードで参照する前にシート名を確認すること
5. 検証前に断言しない；自動化を常に手動作業より優先する
6. 「問題なし」と判断する前に必ず細矢さんの画像と数値を両方確認する
7. 原因が特定できていない状態で「直ります」と言わない
8. 指示「手順とコードを提出」→余計な操作をしない
9. Colabコードには説明文・全角記号を混入させない（SyntaxErrorの原因）
10. コミット確認はRAW（キャッシュあり）ではなくAPI（リアルタイム）を使う
11. 引き継ぎ書は必ず1ファイル・一括で出力する（分割禁止）
12. predict_recordシートは3行構造（ヘッダー/サブヘッダー/データ）
    data=rows[2:]が正しい（rows[1:]はサブヘッダーを含むため誤り）
13. GitHubブラウザJS書き込みはClassicトークンが必要
    Fine-grainedトークンはCORSでPUT操作がブロックされる
14. バックテストのlook-ahead biasに注意
    「現在スコアで過去リターン検証」は先読みバイアスがある
15. 書記官は更新前に必ず前版を取得・文字数確認してから上書きする
    新版文字数 >= 旧版文字数 を必ず守る（2026/03/25 教訓）
