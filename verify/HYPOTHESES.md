# 予測モデル 仮説台帳（HYPOTHESES）

**目的**: 立てた仮説と「なぜその値か」の根拠を記録し、**次回の検証（ic_report の IC）時に必ず読んで当否を判定・調整する**（CEO指示 2026-06-07）。
**読むタイミング**: `monthly_learning` の `verify/ic_report.py` 出力（`IC_レポート` シート / ログ）を見るとき、本書の「現行仮説」と IC を突き合わせる。ic_report は本書への参照行・参照ログを毎回出力する。
**原則**: 仮説は「決め打ち」でなく検証対象。IC が当否を語る。修正は **1度に1パラメータ**。100%級の高勝率・低標本での「IC有意」は偽陽性として弾く。

## H-perstock-0607（2026-06-07 制定・status: 検証中）

### 仮説
短期(1年)/中期(3年)の予測を、市場値の全銘柄貼付でなく**各銘柄の因子**で算出すれば、銘柄ごとの傾向が forward 検証で観測でき、学習で精度を上げられる。

- 短期 `sc_s = 市場short * 0.4 + 銘柄割安度(s3) * 0.6`
- 中期 `sc_m = 市場mid  * 0.4 + 銘柄トレンド(s2) * 0.6`
- 実装: `core/scoring.py: perstock_short / perstock_mid`、呼び出し `learning_batch_monthly.py` Step3。

### 因子の厳密な定義（CEO指示 2026-06-07: ここが曖昧だと次回検証が無駄になる）
出典: `learning_batch_monthly.py: calc()` ＋ `core/config.py` の閾値。スコアは全て 0–100。

- **s2 = トレンド（中期 3年 の銘柄因子）** = `round( thr_high(ROE_slope, RS_THR)*0.6 + thr_high(FCR_slope, FS_THR)*0.4 )`
  - `ROE_slope` = **直近4期の ROE の線形回帰の傾き（%/年）** ＝ `polyfit(range(n), ROE[-4:], 1)` の傾き。ROE = 当期純利益 ÷ 自己資本 ×100。
  - `FCR_slope` = 直近4期の FCR の傾き（%/年）。FCR = FCF ÷ 当期純利益 ×100、FCF = 営業CF + 投資CF。
  - 閾値: `RS_THR=[(4.0,100),(2.0,82),(0.5,64),(0,64),(-0.5,46),(-2.0,28),(-999,12)]`（単位 %/年）、`FS_THR=[(8.0,100),(4.0,80),(0.0,60),(-4.0,40),(-8.0,20),(-999,8)]`。
  - ⚠ **「銘柄トレンド」＝株価モメンタムではない。会計指標(ROE/FCR)の年次の改善トレンド**。データ源=yfinance 年次財務。
- **s3 = 価格/割安度（短期 1年 の銘柄因子）** = `round( thr_low(PEG, PEG_THR)*0.5 + thr_high(FCF利回り, FCY_THR)*0.5 )`
  - `PEG` = PER ÷ (EPS成長率 ×100)。PER=trailingPE、EPS成長率=info.earningsGrowth（無ければ純利益のCAGR）。
  - `FCF利回り` = |FCF| ÷ 時価総額 ×100 (%)。
  - 閾値: `PEG_THR=[(0.5,100),(0.8,85),(1.0,72),(1.2,58),(1.5,42),(2.0,26),(999,12)]`（低いほど良・thr_low）、`FCY_THR=[(8,100),(6,85),(4,70),(3,55),(2,38),(1,22),(0,8)]`。
  - ⚠ **静的バリュエーションのスナップショット（割安度）。価格の値動きではない**。
- 参考 **s1 = Real ROIC（質・長期主軸）** = `round( thr_high(ROE平均, ROE_THR)*0.6 + thr_high(FCR平均, FCR_THR)*0.4 )`。
- **市場値**（market_short / market_mid）= `週次シグナル` 最終行の短期/中期スコア（MacroPhase 由来・**全銘柄共通の単一値**・既定 短期29 / 中期25）。

→ **次回検証は「s2(会計トレンド) が中期3年リターンを予測するか」「s3(割安度) が短期1年リターンを予測するか」を IC で測っている**、と明確に解釈すること（因子の取り違え＝検証の無駄を防ぐ）。

### なぜ 0.4 / 0.6 なのか（CEO質問 2026-06-07・正直な根拠）
**結論: 0.6 は「最適化で導いた値」ではなく、2つの制約を満たす最小限明快な“過半”**。
1. **>0.5 でなければならない理由**: per-stock を市場値より優位にしないと、旧方式(市場0.5＋粗いバンド丸め)のように銘柄差が潰れ「全銘柄同一(Top75全部一緒)」が再発する。銘柄因子が“過半”を握る必要 → 0.5 超。
2. **<1.0（=市場を0にしない）理由**: 1〜3年リターンは相場全体(レジーム)にも実際に左右される。市場成分を完全に消すと相場急変時に個別因子だけで方向を出して外す。市場を 0.4 残す。
3. **なぜ 0.7 や 0.65 でなく 0.6 か**: (1)(2)を満たす範囲(おおむね 0.55〜0.75)の中で、丸く・説明しやすく・市場成分を 1/3 強(0.4)残す控えめな“過半”として 0.6 を選択。**この値の一桁目に理論的必然はない＝だから固定値でなく「開始仮説」**。
4. **正しい値は IC が決める**: 満期後、短期/中期の IC・Q5-Q1・単調性を見て、銘柄成分を増やす(例 0.7)か減らす(例 0.5)かを **1度に1パラメータ**で調整。`core/scoring.py: PERSTOCK_W_MARKET_SHORT / _MID` の1定数を変えるだけ。

### 検証方法（次回 IC レビューでこの節を読む）
- `IC_レポート` の **IC(騰落率)・Q5-Q1・単調性** を軸別に確認。判定: IC≥0.05「有効」/ IC<0「逆相関・要レビュー」/ それ以外「弱い・観察継続」。
- **満期**: 短期は記録から 1 年（最短 2027/03〜）、中期 3 年（2029〜）。`MIN_SAMPLES=20` 未満は判定保留。
- **調整指針（1度に1パラメータ）**:
  - 短期 IC が弱い → ① `PERSTOCK_W_MARKET_SHORT` を下げ銘柄成分を増やす ② 因子割当を見直す（短期↔モメンタム等・要データ追加）。
  - 中期 IC が弱い → `PERSTOCK_W_MARKET_MID` 調整 or 因子見直し。
  - 良好 → 維持。
- **偽陽性ガード**: 低標本での「IC有意」は多重比較由来の偽陽性として採用しない。
