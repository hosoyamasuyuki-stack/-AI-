## バグ修正

### Bug 1: J-Quants V2 master APIスキーマ変更
- **現象**: 銘柄管理が 3/26, 3/30, 4/1, 4/17 の4回連続 failure
- **原因**: `CompanyName`→`CoName`, `Sector33CodeName`→`S33Nm` にAPIが変更
- **対策**: 新旧キーフォールバック（manage_stock.py:58-60）
- **コミット**: 38555d5

### Bug 2: 長期ランクソートが S→D→C→B→A
- **現象**: S の次に A ではなく D が並ぶ
- **原因**: `localeCompare` でアルファベット順（A(65)<B<C<D<S(83)）
- **対策**: `RANK={S:4,A:3,B:2,C:1,D:0}` マップで数値比較
- **コミット**: 8ffdcf9

### Bug 3: v4.3スコア暴落（4792: 57.3/B → 2.8/D）
- **現象**: manage_stock 実行時 57.3/B、その後 dashboard で 2.8/D
- **根本原因**: 4段階の連鎖バグ
  1. 初回 manage_stock（PEG thr_high誤用 + 列順違い）でコアスキャン_v4.3 に列シフト書込
  2. remove→re-add した際、remove_stock がコアスキャンを削除せず古い行が残留
  3. daily_price_update がコアスキャンから 変数1=0.68 を読み int()=0
  4. total=0*0.4+0*0.35+11*0.25=2.75→2.8 を SYNC_COLS で 監視銘柄に上書き
- **対策**:
  - PEG 計算を thr_low に修正
  - manage_stock の new_row を value_map + ヘッダー名ベースに
  - コアスキャン_v4.3 既存行は find_row 検出時も value_map で上書き
  - daily_price_update の変数1/2 ソースを 保有+監視 シート優先に
  - サニティチェック（abs<1.0 で異常検知）
- **コミット**: 3f09b42, b149d14

### Bug 4: 短期・中期列が全銘柄一律
- **現象**: 189銘柄全てに「71点/down/down」と同じ値
- **根本原因**: generate_dashboard が `SHORT_SCORE`/`MID_SCORE`（マクロ共通値）を TD に一律表示
- **対策**: 予測記録シートから各銘柄の方向予測を読み出し、TD に反映
- **コミット**: e0e741c

### Bug 5: verify ワークフロー失敗（4/6, 4/13, 4/15）
- **現象**: 4/15 STEP0検証が PYTHONPATH 欠落で 3回連続 failure
- **原因**: 2026/03/31 のファイル整理で verify/ に移動した際 sys.path 未設定
- **対策**: verify.yml に `PYTHONPATH: ${{ github.workspace }}` 追加
- **コミット**: b234e60
